<task type="auto">
  <name>Task 1: Build arXiv-compliant tarball</name>
  <files>paper/main.bbl, arxiv/cf-libs-gpu.tar.gz</files>
  <action>
    Prepare the paper for arXiv submission:

    1. Generate the .bbl file from the existing paper:
       ```
       cd paper && pdflatex main.tex && bibtex main && pdflatex main.tex && pdflatex main.tex
       ```
       This produces main.bbl with resolved bibliography entries.

    2. Create a flattened version of main.tex for arXiv:
       - Replace all \input{sections/X} with the actual file contents (arXiv handles \input but flattening is more reliable)
       - Replace \bibliography{refs} + \bibliographystyle{elsarticle-harv} with \input{main.bbl} (arXiv does not run BibTeX)
       - Ensure all figure paths are relative and correct (figures/figNN_*.pdf)
       - Keep the elsarticle document class (arXiv supports it)
       - Remove any local-only packages or paths

    3. Build the tarball containing ONLY:
       - main.tex (flattened, self-contained)
       - main.bbl (bibliography)
       - figures/*.pdf (all 7 figure PDFs)

    4. Verify tarball does NOT contain:
       - .aux, .log, .out, .synctex.gz, .fdb_latexmk files
       - .bib file (arXiv uses .bbl, not .bib)
       - .git directory or any dotfiles
       - Python scripts or benchmark data
       - slides/ directory

    The tarball should extract to a flat structure (no nested directory), which arXiv prefers.
  </action>
  <verify>
    1. tar -tzf arxiv/cf-libs-gpu.tar.gz shows only: main.tex, main.bbl, figures/*.pdf
    2. No forbidden files (.aux, .log, .bib, .git, scripts/)
    3. Total tarball size < 10 MB (arXiv limit)
    4. Figure count in tarball matches figure count in paper (7 PDFs)
  </verify>
  <done>arXiv tarball built with flattened LaTeX, .bbl, and 7 figure PDFs; no forbidden files; under 10MB</done>
</task>

<task type="auto">
  <name>Task 2: Test standalone compilation</name>
  <files>arxiv/cf-libs-gpu.tar.gz</files>
  <action>
    Verify the tarball compiles in isolation:

    1. Create a temporary directory
    2. Extract the tarball into it
    3. Run pdflatex main.tex (twice for cross-references)
    4. Verify:
       - Zero errors in pdflatex output
       - PDF is produced
       - All figures are included (no "File not found" warnings)
       - Page count matches expected (~15-25 pages for JQSRT format)
       - No undefined references or citations

    If compilation fails, fix the issue in the tarball contents and rebuild.

    Clean up the temporary directory after verification.
  </action>
  <verify>
    1. pdflatex exits cleanly (zero errors) from extracted tarball
    2. No "undefined reference" or "Citation undefined" warnings
    3. No "File `figures/...' not found" warnings
    4. PDF page count is reasonable (15-25 pages)
  </verify>
  <done>Standalone compilation verified -- tarball produces complete PDF with zero errors from clean extraction</done>
</task>
