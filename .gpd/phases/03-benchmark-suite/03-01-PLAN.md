  <name>Task 1: Common benchmark utilities and Voigt throughput benchmark (BENCH-01)</name>
  <files>benchmarks/common.py, benchmarks/bench_voigt.py</files>
  <action>
    Create benchmarks/common.py with shared utilities:

    1. `hardware_metadata()` -> dict with:
       - GPU model (via jax.devices()[0].device_kind or nvidia-smi parsing)
       - Driver version, CUDA version, JAX version, Python version
       - CPU model (platform.processor())
       - Timestamp (ISO 8601)
       - JAX_PLATFORMS environment variable
    2. `TimingContext` context manager that:
       - Uses time.perf_counter() for wall-clock timing
       - Calls jax.block_until_ready() on the result if result is a JAX array
       - Records individual run times in a list
    3. `benchmark_function(fn, args, n_warmup=3, n_runs=10)` -> dict with mean, std, min, max, all_times
       - Runs fn(*args) n_warmup times (excluded from timing)
       - Then runs fn(*args) n_runs times with TimingContext
       - Calls block_until_ready() on the return value of fn
    4. `save_results(results: dict, output_path: str)` -> writes JSON with indent=2
    5. `ensure_output_dir(path)` -> creates directory if needed

    Create benchmarks/bench_voigt.py:

    1. Import voigt_spectrum_jax from cflibs.radiation.profiles
    2. Import scipy.special.wofz as CPU baseline
    3. Define grid_sizes = [100, 500, 1000, 5000, 10000, 50000, 100000] (wavelength grid points)
    4. For each grid_size:
       a. Create wavelength grid: wl = np.linspace(200, 600, grid_size) [nm]
       b. Create test line parameters: 10 lines with center wavelengths uniformly in [250, 550] nm,
          sigma = 0.01 nm (Doppler), gamma = 0.02 nm (Stark HWHM)
       c. CPU baseline: time scipy.special.wofz on equivalent complex z array (N_wl x N_lines)
       d. GPU (JAX): time voigt_spectrum_jax(wl, centers, sigma, gamma)
       e. Record throughput = N_wl * N_lines / time [evaluations/sec]
    5. Accuracy measurement: for each grid_size, compute max |V_jax - V_scipy| / |V_scipy| (relative error)
    6. Output JSON structure:
       ```json
       {
         "benchmark": "voigt_throughput",
         "hardware": {...},
         "parameters": {"grid_sizes": [...], "n_lines": 10, "n_runs": 10},
         "results": {
           "cpu_throughput": [...],
           "gpu_throughput": [...],
           "cpu_time_mean": [...],
           "gpu_time_mean": [...],
           "cpu_time_std": [...],
           "gpu_time_std": [...],
           "max_relative_error": [...],
           "speedup": [...]
         }
       }
       ```
    7. Script must be runnable as: `python benchmarks/bench_voigt.py --output benchmarks/results/voigt_results.json`
    8. Include --cpu-only flag for testing without GPU
    9. Print summary table to stdout during execution

    Use float64 throughout (per project convention). The accuracy measurement is essential -- it feeds into VALD-01 and Figure 2.
  </action>
  <verify>
    1. `python benchmarks/bench_voigt.py --cpu-only --output /tmp/test_voigt.json` produces valid JSON
    2. JSON contains all required keys: benchmark, hardware, parameters, results
    3. All timing values are positive floats
    4. Throughput values are computed correctly: N_wl * N_lines / mean_time
    5. Relative error values are computed (may be large on CPU-only since both use similar paths, but format is correct)
    6. Script handles JAX not available gracefully (skips GPU benchmarks)
  </verify>
  <done>benchmarks/common.py and benchmarks/bench_voigt.py exist, produce valid JSON output on CPU, timing and accuracy measurements are structurally correct</done>

<task type="auto">
  <name>Task 2: Boltzmann fitting time benchmark (BENCH-02)</name>
  <files>benchmarks/bench_boltzmann.py</files>
  <action>
    Create benchmarks/bench_boltzmann.py:

    1. Import batched_boltzmann_fit from cflibs.inversion.boltzmann_jax
    2. Import numpy for CPU baseline (np.polyfit in a loop)
    3. Define element_counts = [1, 2, 3, 5, 7, 10, 15, 20] (number of elements in batch)
    4. Define lines_per_element = [10, 25, 50, 100] (number of spectral lines per element)
    5. For each (element_count, lines_per_element) combination:
       a. Generate synthetic Boltzmann plot data:
          - E_k values: uniform random in [0, 10] eV (sorted per element)
          - y = ln(I*lambda/gA) values: slope * E_k + intercept + noise
          - slope = -1/(k_B * T) with T randomly in [8000, 15000] K per element
          - intercept randomly in [10, 20]
          - noise ~ N(0, 0.1) (realistic scatter)
          - weights = 1/noise^2
       b. CPU baseline: time a loop of np.polyfit(E_k, y, 1) calls, one per element
       c. GPU (JAX): time batched_boltzmann_fit(E_k_batch, y_batch, w_batch, N_valid_batch)
          where data is padded to max lines_per_element
       d. Record wall time (ms) for each
    6. Output JSON structure:
       ```json
       {
         "benchmark": "boltzmann_fitting",
         "hardware": {...},
         "parameters": {"element_counts": [...], "lines_per_element": [...], "n_runs": 10},
         "results": [
           {
             "element_count": N,
             "lines_per_element": M,
             "cpu_time_ms_mean": ...,
             "cpu_time_ms_std": ...,
             "gpu_time_ms_mean": ...,
             "gpu_time_ms_std": ...,
             "speedup": ...
           }
         ]
       }
       ```
    7. Runnable as: `python benchmarks/bench_boltzmann.py --output benchmarks/results/boltzmann_results.json`
    8. Include --cpu-only flag
    9. Print summary table to stdout

    IMPORTANT: The CPU baseline must use np.polyfit in a Python loop (the actual current approach), NOT a vectorized NumPy solution. We are comparing the real CPU codepath against the JAX batched kernel.
  </action>
  <verify>
    1. `python benchmarks/bench_boltzmann.py --cpu-only --output /tmp/test_boltzmann.json` produces valid JSON
    2. JSON contains element_count x lines_per_element result entries
    3. CPU timing grows roughly linearly with element_count (loop overhead)
    4. All timing values are positive
    5. Speedup = cpu_time / gpu_time computed correctly
  </verify>
  <done>benchmarks/bench_boltzmann.py exists, produces valid JSON with timing for all element_count x lines_per_element combinations, CPU baseline uses realistic loop approach</done>
</task>
