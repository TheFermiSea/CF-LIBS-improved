<task type="auto">
  <name>Task 1: End-to-end pipeline benchmark (BENCH-06)</name>
  <files>benchmarks/bench_e2e_pipeline.py</files>
  <action>
    Create benchmarks/bench_e2e_pipeline.py:

    This benchmark measures the COMPLETE forward model pipeline time, broken down by component, for CPU vs GPU. It directly addresses the forbidden proxy fp-isolated-kernel.

    1. Define the pipeline stages and their implementations:
       a. **Stage 1 - Data preparation**: Load atomic data, create wavelength grid, prepare plasma parameters
       b. **Stage 2 - Saha-Boltzmann equilibrium**: Compute ionization fractions and level populations
          - CPU: existing Picard iteration via cflibs.plasma solver
          - GPU: Anderson-accelerated solver from anderson_solver.py
       c. **Stage 3 - Boltzmann fitting** (for the inversion direction):
          - CPU: np.polyfit loop
          - GPU: batched_boltzmann_fit from boltzmann_jax.py
       d. **Stage 4 - Voigt profile computation**:
          - CPU: scipy.special.wofz-based profile
          - GPU: voigt_spectrum_jax from profiles.py
       e. **Stage 5 - Spectrum assembly**: Sum line contributions, apply instrument response
          - CPU: numpy operations
          - GPU: JAX operations (part of batch_forward_model)
       f. **Stage 6 - Closure** (for inversion):
          - CPU: standard normalization
          - GPU: softmax_closure from softmax_closure.py

    2. Define batch_sizes:
       - Full mode: [1, 10, 100, 1000, 10000]
       - Small mode (--small): [1, 10, 100]

    3. Define test scenario:
       - 5 elements (Fe, Cu, Ni, Cr, Mn representative of steel LIBS)
       - 50 lines total (10 per element)
       - Wavelength grid: 4096 points in [200, 600] nm
       - T ~ 10000 K, n_e ~ 1e16 cm^-3 (typical LTE plasma)
       - Use synthetic atomic data (self-contained, no DB dependency)

    4. For each batch_size:
       a. **CPU pipeline**: Run stages 1-6 sequentially in a Python loop over batch elements.
          Time each stage separately using time.perf_counter().
          Record: stage_times_cpu = {stage_name: time_ms for each stage}
          Total CPU time = sum of stage times

       b. **GPU pipeline**: Run batch_forward_model (stages 1-5 combined via vmap).
          Time the overall call (with block_until_ready).
          Time data transfer separately (device_put).
          For component breakdown, also time individual stages when called separately on GPU:
          - Saha stage alone
          - Voigt stage alone
          - Assembly stage alone
          Record: stage_times_gpu, total_gpu_time, transfer_time

       c. Compute: speedup = cpu_total / gpu_total, speedup_no_transfer = cpu_total / gpu_compute_only

    5. Output JSON:
       ```json
       {
         "benchmark": "end_to_end_pipeline",
         "hardware": {...},
         "parameters": {
           "batch_sizes": [...], "n_elements": 5, "n_lines": 50,
           "n_wl": 4096, "n_runs": 5
         },
         "results": [
           {
             "batch_size": B,
             "cpu_total_ms_mean": ..., "cpu_total_ms_std": ...,
             "gpu_total_ms_mean": ..., "gpu_total_ms_std": ...,
             "gpu_compute_ms_mean": ...,
             "gpu_transfer_ms_mean": ...,
             "speedup": ...,
             "speedup_no_transfer": ...,
             "cpu_component_breakdown": {
               "data_prep": ..., "saha_boltzmann": ...,
               "boltzmann_fit": ..., "voigt_profiles": ...,
               "assembly": ..., "closure": ...
             },
             "gpu_component_breakdown": {
               "transfer": ..., "saha_boltzmann": ...,
               "voigt_profiles": ..., "assembly": ..., "total_compute": ...
             },
             "crossover_note": "GPU faster/slower than CPU at this batch size"
           }
         ],
         "summary": {
           "crossover_batch_size": "estimated batch size where GPU overtakes CPU",
           "max_speedup": ...,
           "max_speedup_batch_size": ...,
           "bottleneck_cpu": "component name",
           "bottleneck_gpu": "component name"
         }
       }
       ```
    6. Runnable as: `python benchmarks/bench_e2e_pipeline.py --output benchmarks/results/e2e_results.json`
    7. Flags: --small, --cpu-only, --output PATH
    8. Print summary to stdout: table of batch_size vs CPU time vs GPU time vs speedup

    CRITICAL: This script must NOT sum individual kernel benchmark results. It must run the actual pipeline end-to-end, including all overhead (memory allocation, synchronization, data movement). This is what distinguishes it from individual kernel benchmarks and prevents the fp-isolated-kernel forbidden proxy.
  </action>
  <verify>
    1. `python benchmarks/bench_e2e_pipeline.py --small --cpu-only --output /tmp/test_e2e.json` produces valid JSON
    2. Component breakdown times sum to within 10% of total time (overhead from timer calls)
    3. CPU total time increases roughly linearly with batch_size (sequential processing)
    4. Summary section is populated with crossover estimate and bottleneck identification
    5. All timing values are positive
    6. Script does NOT reference output from other benchmark scripts (it is self-contained)
  </verify>
  <done>benchmarks/bench_e2e_pipeline.py produces valid JSON with total pipeline timing and component breakdown for all batch sizes, component times sum to approximately total, crossover batch size estimated</done>
