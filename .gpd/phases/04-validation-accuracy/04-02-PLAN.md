<objective>
Validate GPU vs CPU pipeline parity on real LIBS data with known compositions (VALD-06). Two datasets: Aalto University mineral library (61 spectra with qualitative compositions from mineral formulas) and ChemCam CCCT calibration targets (6 targets with certified elemental weight fractions from Fabre et al. 2011).

Purpose: Demonstrate that GPU acceleration produces identical results on real experimental data, and that the CF-LIBS pipeline recovers known compositions at expected accuracy levels. This is the "real-world" complement to the numerical accuracy tests in Plan 04-01.
Output: GPU vs CPU comparison scripts for both datasets, combined validation report with paper-ready tables.
</objective>

<task type="auto">
  <name>Task 1: Aalto mineral GPU vs CPU parity benchmark (VALD-06a)</name>
  <files>validation/real_data/run_aalto_gpu_vs_cpu.py</files>
  <action>
    Create a validation script that runs the CF-LIBS inversion pipeline on Aalto mineral spectra using both GPU (JAX) and CPU (numpy/scipy) codepaths, then compares results.

    **Data loading:**
    1. Load all 61 mineral spectra from data/aalto_libs/minerals/ (CSV files: wavelength_nm, intensity columns)
    2. Load mineral compositions from the MINERAL_COMPOSITIONS dict in scripts/run_aalto_benchmark.py (qualitative: element presence, not quantitative weight fractions)
    3. Also load 13 pure element spectra from data/aalto_libs/elements/ for pure-element validation

    **Pipeline design:**
    The key comparison is: given IDENTICAL input spectra, do GPU and CPU codepaths produce identical outputs?

    Build two pipeline variants that share preprocessing but differ in the computational kernels:
    - **CPU pipeline:** Uses existing numpy/scipy codepaths (voigt_profile from profiles.py, boltzmann_fit from boltzmann.py, simple Picard iteration, standard closure)
    - **GPU pipeline:** Uses JAX codepaths (voigt_spectrum_jax, batched_boltzmann_fit, anderson_solve, softmax_closure)

    For each of the 61 mineral spectra:
    1. Load and preprocess spectrum identically for both pipelines
    2. Run line detection and identification (shared -- this is not GPU-accelerated)
    3. Run Boltzmann fitting: CPU (np.polyfit per element) vs GPU (batched_boltzmann_fit)
    4. Run Saha-Boltzmann solver: CPU (Picard iteration) vs GPU (Anderson acceleration)
    5. Run closure: CPU (standard sum-to-1 normalization) vs GPU (softmax closure)
    6. Compare outputs: T, n_e, detected elements, concentrations

    **Parity metrics:**
    - Temperature: |T_gpu - T_cpu| / T_cpu (expect < 0.1% since same math, different implementation)
    - Electron density: |n_e_gpu - n_e_cpu| / n_e_cpu
    - Element detection: set(elements_gpu) == set(elements_cpu)
    - Concentrations: max |C_gpu[i] - C_cpu[i]| / C_cpu[i] for each element

    **Element detection accuracy (vs mineral formula):**
    - For each mineral: what fraction of expected elements (from MINERAL_COMPOSITIONS) are detected?
    - Recall = |detected AND expected| / |expected|
    - Precision = |detected AND expected| / |detected|
    - Note: O cannot be detected by LIBS (no strong O emission in typical LIBS range), so exclude O from expected elements

    **Output:** JSON with per-spectrum results including all parity metrics and element detection accuracy.

    Note: If the full inversion pipeline requires the atomic database (ASD_da/libs_production.db), check for its existence. If not available, use a reduced test that exercises only the mathematical kernels (Boltzmann fitting, Saha solver, closure) on pre-extracted line data rather than the full pipeline. Document which path was taken.

    The script MUST be runnable with: `python validation/real_data/run_aalto_gpu_vs_cpu.py --db ASD_da/libs_production.db --output validation/real_data/results/aalto_results.json`
  </action>
  <verify>
    1. GPU and CPU pipelines produce identical element detection for all 61 mineral spectra
    2. T and n_e agree to < 0.1% between GPU and CPU for all converged spectra
    3. Concentration parity: max relative difference < 0.1% between GPU and CPU
    4. Element detection recall > 50% averaged across minerals (expected given instrument mismatch and O exclusion)
    5. Pure element spectra: correct element detected as dominant in > 10/13 cases
    6. Script produces valid JSON output
    7. No NaN/Inf in any output quantity
  </verify>
  <done>Aalto validation script demonstrates GPU-CPU parity on 61 real mineral spectra. Element detection recall and parity metrics computed and stored in JSON. GPU and CPU produce identical results to within numerical precision.</done>
