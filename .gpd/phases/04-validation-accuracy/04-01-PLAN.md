  <action>
    Create three independent validation scripts. Each must be runnable standalone and produce JSON output.

    **VALD-01: Voigt Accuracy (test_voigt_accuracy.py)**
    1. Import voigt_spectrum_jax from cflibs.radiation.profiles and scipy.special.wofz as CPU reference
    2. Define parameter grid: x = np.linspace(0, 50, 20), y = np.logspace(-6, np.log10(50), 20) -- 400 points total
    3. For each (x, y): compute Voigt via JAX kernel and scipy reference. Record relative error = |V_jax - V_ref| / |V_ref|
    4. Also test edge cases: x=0 (pure Lorentzian), y=0 (pure Gaussian limit), x=y=0, x=1000 (far wing), y=1e-10 (extremely narrow Lorentzian)
    5. Output: max error, mean error, error heatmap data, edge case results
    6. PASS criterion: max relative error < 1e-6 across ALL test points, no NaN/Inf

    Note on Voigt parameterization: the JAX kernel takes (wavelengths, centers, sigma, gamma) in nm units. To sweep the standard (x,y) Faddeeva parameter space, convert: x = (lambda - lambda_0) / (sigma * sqrt(2)), y = gamma / (sigma * sqrt(2)). Fix sigma=0.01 nm, compute gamma = y * sigma * sqrt(2), lambda = lambda_0 + x * sigma * sqrt(2). Compare the profile VALUE (not the integrated area) at each point.

    **VALD-02: Boltzmann Accuracy (test_boltzmann_accuracy.py)**
    1. Import batched_boltzmann_fit from cflibs.inversion.boltzmann_jax and np.polyfit as CPU reference
    2. Generate 1000 random Boltzmann datasets:
       - T = uniform(5000, 30000) K -> slope = -1/(k_B * T) in eV^-1
       - N_lines per element: uniform(5, 50)
       - N_elements: choices from [1, 2, 5, 10, 20]
       - E_k (upper level energies): uniform(0, 10) eV
       - y = slope * E_k + intercept + noise(sigma=0.1)
       - Weights: uniform(0.1, 10) per line
    3. Run batched_boltzmann_fit on padded batch. Run np.polyfit element-by-element as reference
    4. Compare: slope relative error, intercept relative error, R^2 difference
    5. PASS criterion: max |slope_jax - slope_ref| / |slope_ref| < 1e-10, same for intercept

    **VALD-04: Softmax Accuracy (test_softmax_accuracy.py)**
    1. Import softmax_closure, inverse_softmax from cflibs.inversion.softmax_closure
    2. Generate 1000 random theta vectors: D = random choice from [2,3,5,10,15,20], theta = randn(D) * scale where scale = random choice from [0.1, 1.0, 10.0, 100.0]
    3. For each: compute C = softmax(theta), check abs(sum(C) - 1)
    4. Round-trip: verify abs(softmax(inverse_softmax(C)) - C) < 1e-14 element-wise
    5. Edge cases: theta = [500, -500, 0] (extreme), theta = [0] (single element), theta = zeros(D) (uniform), theta with one element >> others (near-degenerate)
    6. Gradient check: jax.grad through softmax produces finite values (no NaN)
    7. PASS criterion: abs(sum(C) - 1) < 1e-15 for all cases, no NaN/Inf in any edge case
  </action>
  <verify>

    3. VALD-04: All 1000 cases + edge cases have sum-to-1 error < 1e-15
    4. No NaN or Inf in any output across all three scripts
    5. Each script produces valid JSON with structured results
  </verify>
