    3. **paper/sections/methods.tex**: ~4-5 pages covering all 5 optimization targets:

       **Section 2.1: GPU-Optimized Voigt Profile**
       - Voigt as convolution of Gaussian (Doppler) and Lorentzian (Stark + natural)
       - Faddeeva function: V(lambda) = Re[w(z)] / (sigma_G * sqrt(2*pi))
       - Weideman N=36 rational approximation: branch-free, 236 FLOPs/eval, <1e-13 error (float64)
       - Cite Weideman (1994), compare briefly with Humlicek W4 (branching incompatible with JAX AD) and Zaghloul 2024 (accuracy reference)
       - JAX parallelization: broadcasting outer product over (N_wl, N_lines) grid

       **Section 2.2: Vectorized Boltzmann Fitting**
       - Boltzmann plot: y = ln(I*lambda/(g_k*A_ki)) vs x = E_k, slope = -1/(k_B*T)
       - Closed-form WLS: 5 dot products per element (S_w, S_wx, S_wy, S_wxx, S_wxy)
       - Pad-and-mask batching for variable line counts across elements
       - 10*N_max + 7 FLOPs per element

       **Section 2.3: Anderson-Accelerated Saha-Boltzmann Solver**
       - Saha-Boltzmann as fixed-point: g(n_e) = charge_balance(T, C, n_e) -> n_e
       - Anderson acceleration: store m previous iterates, solve LS for mixing coefficients
       - Cite Walker & Ni (2011), Evans et al. (2018) for convergence theory
       - JAX implementation via lax.scan with configurable depth m

       **Section 2.4: Softmax Closure**
       - Composition constraint: sum(C_i) = 1, C_i > 0
       - Softmax: C_i = exp(theta_i) / sum(exp(theta_j)) with log-sum-exp stabilization
       - Jacobian: J_ij = C_i(delta_ij - C_j)
       - Enables gradient-based optimization of compositions

       **Section 2.5: Batch Forward Model**
       - Four-stage pipeline: Saha -> Boltzmann -> Emissivity -> Voigt+Assembly
       - vmap decomposition: single-level over batch (T, n_e, C), broadcasting for profiles
       - Memory budget: B_max = floor((M_GPU - overhead) / (N_wl * N_lines * dtype_bytes))
       - Cite ExoJAX for comparison of JAX-based spectral batch approaches
