    1. The fixed-point map g(log_ne) -> log_ne_new:
       - Given log_ne, T_eV, and compositions C_i, compute Saha ionization ratios for each element
       - Sum electron contributions: n_e_new = sum over elements and stages of z * n_z(T_eV, n_e, C_i)
       - Return log(n_e_new)

       The Saha computation should use the SAME physics as saha_boltzmann.py but in pure JAX:
       - Saha ratio: S = (2 * U_{z+1} / (n_e * U_z)) * SAHA_CONST_CM3 * T_eV^1.5 * exp(-chi_z / T_eV)
       - IPD via Debye-Huckel: delta_chi_eV (import from existing code or reimplement in JAX)
       - Partition functions: use polynomial_partition_function_jax from cflibs.plasma.partition

    2. Anderson acceleration core (per DERV-03, Walker-Ni unconstrained formulation):
       - Maintain circular buffer of m+1 residuals and g-values
       - At step k with history depth mk = min(k, m):
         * Build Delta_R matrix from residual differences
         * Solve gamma = argmin ||r_k - Delta_R @ gamma||^2 + lambda * ||gamma||^2 via Tikhonov-regularized normal equations
         * Update: x_{k+1} = g(x_k) - (Delta_G + Delta_R) @ gamma
       - Use jax.lax.while_loop for JIT compatibility (NOT Python for loop)
       - Convergence criterion: |r_k| / |x_k| < tol (default 1e-10)
       - Max iterations: 50 (configurable)

    3. Safeguarding:
       - Clamp log_ne to [log(1e12), log(1e20)] after each update
       - If residual increases by >10x, restart AA history (reset mk=0)
       - Fallback: if m=0, this is pure Picard iteration (the m=0 limiting case from DERV-03)

    4. Public API:
       ```python
       @dataclass
       class AndersonSolverResult:
           n_e: jnp.ndarray  # converged electron density [cm^-3]
           iterations: int
           residual: float
           converged: bool
           residual_history: jnp.ndarray  # for diagnostics

       def anderson_solve(
           T_eV: float,
           compositions: jnp.ndarray,  # (N_elements,)
           atomic_data: tuple,  # (ionization_potentials, partition_coeffs, charges, ...)
           n_e_init: float = 1e16,
           m: int = 3,
           tol: float = 1e-10,
           max_iter: int = 50,
           tikhonov_lambda: float = 1e-6,
       ) -> AndersonSolverResult:
       ```

    5. Also provide `picard_solve` with same signature (m=0 wrapper) for comparison.

    1. Core functions (all @jit decorated):

       ```python
       def softmax_closure(theta: jnp.ndarray) -> jnp.ndarray:
           """Convert unconstrained theta to compositions C_i = exp(theta_i) / sum(exp(theta_j)).

           Uses log-sum-exp trick for numerical stability:
           C_i = exp(theta_i - theta_max) / sum(exp(theta_j - theta_max))

           Parameters
           ----------
           theta : array shape (D,) or (B, D)
               Unconstrained parameters
           Returns
           -------
           C : array same shape as theta
               Compositions on the simplex, sum(C) = 1
           """
       ```

       ```python
       def inverse_softmax(C: jnp.ndarray) -> jnp.ndarray:
           """Convert compositions to unconstrained theta = log(C) - mean(log(C)).

           The centering by mean(log(C)) is the conventional choice
           (theta is shift-invariant, this picks the centered representative).
           """
       ```

       ```python
       def softmax_jacobian(theta: jnp.ndarray) -> jnp.ndarray:
           """Analytical Jacobian J_ij = C_i * (delta_ij - C_j).

           Returns (D, D) matrix. Rank D-1.
           """
       ```

    2. Handle both 1D (D,) and batched 2D (B, D) inputs. Use jnp.atleast_2d internally, squeeze output if input was 1D.

    3. Edge cases:
       - D=1: C = [1.0] always (from DERV-04 limiting case)
       - D=2: C = [sigmoid(theta_1 - theta_2), sigmoid(theta_2 - theta_1)] (from DERV-04 limiting case)
       - Near-zero concentrations: softmax naturally handles this (exp(-700) -> 0 in float64)
       - Very large |theta|: log-sum-exp trick prevents overflow

    4. Round-trip consistency:
       softmax_closure(inverse_softmax(C)) should recover C to machine precision
       (up to the inherent shift ambiguity, which centering resolves)
