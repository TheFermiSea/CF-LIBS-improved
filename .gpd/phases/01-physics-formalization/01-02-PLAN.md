  <name>Task 1: Derive Anderson acceleration for Saha-Boltzmann fixed-point (DERV-03)</name>
  <files>.gpd/research/derivations/derv03_anderson_acceleration.tex</files>
  <action>
    Produce a complete LaTeX derivation of Anderson acceleration applied to the Saha-Boltzmann charge-balance iteration. Structure:

    1. PHYSICS MOTIVATION (0.5 pages):
       - The Saha-Boltzmann system couples ionization balance, level populations, and charge neutrality.
       - Given (T, C_i), the electron density n_e is determined self-consistently: the Saha equation gives ionization fractions as functions of (T, n_e), and charge neutrality requires n_e = sum_s z_s * n_s(T, n_e, C_i).
       - This is a fixed-point problem: define g(n_e) = sum_s z_s * n_s(T, n_e, C_i), then the solution satisfies n_e = g(n_e).
       - The current implementation in saha_boltzmann.py uses Picard (simple) iteration: n_e^{k+1} = g(n_e^k). This converges linearly with rate |g'(n_e*)|.
       - State the fixed-point map explicitly using the codebase Saha equation:
         S_z = (SAHA_CONST/n_e) * T_eV^{3/2} * (U_{z+1}/U_z) * exp(-chi_z/T_eV)
         For 3-stage (neutral + singly + doubly ionized):
         n_I = C_s * n_total / (1 + S_1 + S_1*S_2)
         n_II = S_1 * n_I, n_III = S_2 * n_II
         g(n_e) = sum_elements sum_stages z_stage * n_stage(n_e)
       - Note: the dependence on n_e enters through Saha ratios S_z ~ 1/n_e, making g a nonlinear function of n_e.

    2. ANDERSON ACCELERATION ALGORITHM (1.5 pages):
       a. Define the residual: r_k = g(n_e^k) - n_e^k. At the fixed point, r* = 0.
       b. Collect history: at step k with depth m, store the most recent m+1 pairs (n_e^{k-j}, g(n_e^{k-j})) for j = 0, ..., min(k, m).
       c. CONSTRAINED formulation (Anderson 1965, Walker & Ni 2011):
          - Seek mixing coefficients alpha = (alpha_0, ..., alpha_{m_k}) with sum(alpha_j) = 1
          - Minimize ||sum_{j=0}^{m_k} alpha_j * r_{k-j}||_2
          - Update: n_e^{k+1} = sum_{j=0}^{m_k} alpha_j * g(n_e^{k-j})
       d. UNCONSTRAINED formulation (more common in practice, Walker & Ni 2011 Eq. 2.2):
          - Define difference matrices: Delta_R = [r_k - r_{k-1}, ..., r_k - r_{k-m_k}] shape (1, m_k) for scalar n_e
          - Solve: gamma = argmin ||r_k - Delta_R * gamma||_2 (unconstrained LS, m_k unknowns)
          - Define Delta_G = [g_k - g_{k-1}, ..., g_k - g_{k-m_k}]
          - Update: n_e^{k+1} = g(n_e^k) - (Delta_G + Delta_R) * gamma
       e. For scalar n_e (1D problem): Delta_R is a row vector, gamma is a column vector. The LS solve has a closed-form solution via the pseudoinverse.
       f. For the extended state vector x = [log(n_e), log(T)] (2D joint iteration): Delta_R is (2, m_k), solve via QR or normal equations with Tikhonov regularization.

    3. CONVERGENCE ANALYSIS (1 page):
       - Relate Anderson acceleration to DIIS (Direct Inversion in the Iterative Subspace) from quantum chemistry. AA and DIIS are mathematically equivalent for linear problems (Rohwedder & Schneider 2011).
       - State the convergence theorem (Toth & Kelley 2015): If g is Lipschitz continuously differentiable near x* with g(x*) = x*, and the gain theta_k from the LS solve satisfies theta_k <= eta < 1 for all k, then AA converges r-linearly with rate depending on the condition of the residual history matrix.
       - For smooth contractive maps (|g'(x*)| < 1), AA(m) converges at a rate comparable to GMRES(m) applied to the linearized problem (I - g'(x*)) * delta_x = r.
       - Practical implication for Saha-Boltzmann: the Picard convergence rate is |g'(n_e*)|. For typical LIBS conditions (T ~ 1 eV, n_e ~ 1e17), estimate |g'| ~ 0.3-0.7 (moderate contraction). AA with m=3 should reduce from ~15-30 iterations (Picard) to ~5-8 iterations.
       - State conditions under which convergence fails: |g'(n_e*)| >= 1 (non-contractive). This can occur at very high T where multiply-ionized species dominate and the electron contribution is highly nonlinear.

    4. JAX IMPLEMENTATION STRATEGY (1 page):
       - Use jax.lax.scan for the iteration loop (JIT-compatible, no Python overhead).
       - State the scan carry: (n_e, R_history, G_history, k) where R_history and G_history are (m+1,) arrays for the scalar case.
       - Circular buffer indexing: idx = k % (m+1) for history storage.
       - The LS solve for gamma: for m <= 5, use jnp.linalg.lstsq or explicit normal equations. The problem is tiny (m unknowns), so overhead is negligible.
       - Alternative: jax.custom_root for implicit differentiation through the fixed point. This avoids differentiating through the iteration and instead uses the implicit function theorem: dx*/dtheta = (I - g'(x*))^{-1} * dg/dtheta. State this as the RECOMMENDED approach for gradient-based optimization that wraps the solver.
       - Batching: vmap over (T, C_i) parameter batches. Each batch element runs independent AA with independent history. No cross-batch communication.
       - Memory: for m=5, need 2*(m+1) = 12 scalars of history per batch element. Negligible.

    5. SAFEGUARDING (0.5 pages):
       - Clamping: after each update, clamp n_e to [1e12, 1e20] cm^-3 (physical range).
       - Residual monitoring: if ||r_{k+1}|| > 10 * ||r_k|| (divergence detected), restart AA with m=0 (Picard step) and clear history.
       - Tikhonov regularization: replace the LS solve min||r_k - Delta_R * gamma|| with min||r_k - Delta_R * gamma|| + lambda * ||gamma|| where lambda = 1e-6 * ||Delta_R||_F. This prevents ill-conditioning when residual differences are nearly collinear.
       - Damping: n_e^{k+1} = (1-beta) * n_e^k + beta * n_e_AA^{k+1} with beta = 1.0 (full step) by default, reduced to beta = 0.5 if oscillation detected.

    6. LIMITING CASES (0.5 pages):
       - m=0: Anderson reduces to Picard. The LS problem has no unknowns; alpha_0 = 1 trivially. Verify: n_e^{k+1} = g(n_e^k).
       - m=1: For scalar n_e, the single mixing coefficient is gamma = r_k / (r_k - r_{k-1}). The update becomes n_e^{k+1} = g(n_e^k) - gamma * (g(n_e^k) - g(n_e^{k-1}) + r_k - r_{k-1}). Show this is equivalent to the Aitken delta-squared method (secant-like acceleration).
       - Single element: the Saha equation has an analytical solution n_e = f(T, C, n_total). Anderson is unnecessary but should converge in 1 iteration from any starting point if applied (verify).
       - High n_e limit (n_e >> Saha threshold): all species neutral, g(n_e) -> 0 for all n_e. Fixed point at n_e = 0 (unphysical). Must seed with physically motivated initial guess n_e ~ 0.01 * n_total.

    7. VALIDATION CRITERIA:
       - Convergence: AA(m=3) converges to ||r|| < 1e-8 in fewer iterations than Picard for 90% of test cases in the parameter space T in [0.5, 3.0] eV, n_e in [1e15, 1e18] cm^-3.
       - Fixed-point accuracy: |n_e_AA - n_e_Picard| / n_e_Picard < 1e-8 (same fixed point).
       - Safeguarding: no NaN or divergence for any test case in the parameter space.
       - JAX compatibility: the full iteration compiles under jax.jit without Python-level control flow.

    Reference the existing Picard iteration pattern in cflibs/plasma/saha_boltzmann.py (the solve_ionization_balance method) for the fixed-point map definition.

  <name>Task 2: Formalize softmax closure vs ILR transform (DERV-04)</name>
  <files>.gpd/research/derivations/derv04_softmax_closure.tex</files>
  <action>
    Produce a complete LaTeX derivation comparing softmax and ILR parameterizations for the CF-LIBS compositional closure constraint. Structure:

    1. PHYSICS MOTIVATION (0.5 pages):
       - CF-LIBS determines composition from Boltzmann plot intercepts: C_s proportional to U_s(T) * exp(q_s).
       - The closure constraint sum(C_i) = 1 eliminates the unknown experimental factor F.
       - For gradient-based joint optimization (jointly fitting T, n_e, and all C_i), we need a differentiable parameterization that maps unconstrained R^D or R^{D-1} to the simplex Delta^{D-1} = {C in R^D : C_i >= 0, sum C_i = 1}.
       - Two standard approaches: softmax (overparameterized, D -> D) and ILR (minimal, D-1 -> D).
       - Reference the existing implementations: softmax in joint_optimizer.py (lines 17-19 docstring), ILR in closure.py (Helmert basis + ilr_transform/ilr_inverse).

    2. SOFTMAX PARAMETERIZATION (1 page):
       a. Definition: C_i = exp(theta_i) / sum_j exp(theta_j) for theta in R^D (D unconstrained parameters).
       b. Log-sum-exp stabilization: C_i = exp(theta_i - theta_max) / sum_j exp(theta_j - theta_max) where theta_max = max_j(theta_j). This prevents overflow for large |theta|.
       c. Jacobian: dC_i/dtheta_j = C_i * (delta_{ij} - C_j). This is the matrix J = diag(C) - C * C^T.
       d. Rank: J has rank D-1 (the null space is spanned by the all-ones vector, corresponding to the shift symmetry theta -> theta + c).
       e. Condition number: For uniform composition C_i = 1/D, the nonzero singular values of J are all equal to 1/D. For extreme compositions (one C_i -> 1), the smallest nonzero singular value is ~ min(C_i), which can be exponentially small.
       f. Hessian structure: d^2 L / d(theta_i) d(theta_j) via chain rule through softmax. State the form for L-BFGS-B optimizer information.
       g. Inverse: theta_i = log(C_i) + const (the constant is the shift freedom). Unique up to the shift.

    3. ILR (ISOMETRIC LOG-RATIO) PARAMETERIZATION (1 page):
       a. Centered log-ratio (CLR): clr(C)_i = log(C_i) - (1/D) * sum_j log(C_j). Maps simplex to the hyperplane sum(clr_i) = 0 in R^D.
       b. ILR: z = clr(C) @ V where V is the (D, D-1) Helmert contrast matrix. Maps simplex to R^{D-1} (no redundancy).
       c. Helmert basis V: V_{ij} = 1/sqrt(j*(j+1)) for i < j, V_{jj} = -j/sqrt(j*(j+1)), V_{ij} = 0 for i > j. Verify V^T V = I_{D-1}.
       d. Inverse: C = softmax(V @ z) = exp(V @ z) / sum exp(V @ z). Note: the ILR inverse naturally uses the softmax of the CLR coordinates.
       e. Jacobian: dC_i/dz_k = sum_j (dC_i/dclr_j) * V_{jk}. The CLR-to-simplex Jacobian is the same softmax Jacobian as above, composed with the Helmert rotation.
       f. Condition number: For uniform composition, the ILR Jacobian has condition number 1 (isometric by construction). Near boundaries, it inherits the log singularity: as any C_i -> 0, log(C_i) -> -infinity and the Jacobian diverges.
       g. The boundary singularity is WORSE for ILR than for softmax: ILR requires C_i > 0 strictly (log is undefined at 0), while softmax naturally excludes C_i = 0 (exp(theta_i) > 0 always).

    4. COMPARISON TABLE (0.5 pages):
       Build a comparison matrix:

       | Property | Softmax | ILR |
       |----------|---------|-----|
       | Parameters | D (overparameterized) | D-1 (minimal) |
       | Domain | R^D -> Delta^{D-1} | R^{D-1} -> Delta^{D-1} |
       | Boundary behavior | Smooth (C_i -> 0 as theta_i -> -inf) | Singular (log diverges at C_i = 0) |
       | Jacobian rank | D-1 (null space: shift) | D-1 (full rank) |
       | Condition at uniform | sigma_min = 1/D | sigma_min = 1/D (isometric) |
       | Condition at boundary | sigma_min ~ min(C_i) | sigma_min ~ 0 (log singularity) |
       | JIT compatibility | Yes (exp, sum, divide) | Yes (log, matmul, exp, sum) |
       | Autodiff | Clean (no branches) | Clean (no branches) |
       | Implementation | 3 lines (jax.nn.softmax) | 10+ lines (Helmert + CLR + project) |
       | Existing in codebase | joint_optimizer.py | closure.py (ilr_transform, ilr_inverse) |

    5. GRADIENT FLOW ANALYSIS (0.5 pages):
       - For L-BFGS-B optimization of a loss L(C(theta)):
         dL/dtheta_i = sum_j (dL/dC_j) * (dC_j/dtheta_i) = sum_j (dL/dC_j) * C_j * (delta_{ij} - C_i)
       - This is automatically computed by JAX autodiff; derive it here for the paper and for understanding gradient magnitude.
       - Key insight: the gradient magnitude scales as ||dL/dtheta|| ~ max(C_i) * ||dL/dC|| (since J has spectral norm ~ max(C_i)). For typical LIBS compositions (major element at 50-90%), the gradient is well-scaled.
       - For ILR: dL/dz = V^T @ (diag(C) - C C^T) @ (dL/dC). Same scaling but with the Helmert rotation.
       - Practical conclusion: gradient flow is comparable for both; softmax has slightly better conditioning near simplex boundaries.

    6. LIMITING CASES (0.5 pages):
       - D=1: softmax gives C_1 = exp(theta_1)/exp(theta_1) = 1 identically (trivial). ILR: z is empty (0-dimensional), C_1 = 1. Both correct.
       - D=2: softmax gives C_1 = sigma(theta_1 - theta_2) where sigma is the logistic sigmoid. ILR gives z in R^1, C = ilr_inverse(z, 2). Verify these are related by a simple reparameterization.
       - Uniform theta_i = c: softmax gives C_i = 1/D for all i (uniform mixture). ILR: z = 0 maps to C_i = 1/D. Both give the uniform prior.
       - One dominant element (theta_j >> theta_i for i != j): C_j -> 1, all other C_i -> 0 exponentially. Gradient dL/dtheta_j -> 0 (saturation). This is the vanishing gradient regime. For LIBS with 2-15 elements at 0.1-90% concentration, this is unlikely to be a practical issue.

    7. RECOMMENDATION AND VALIDATION (0.5 pages):
       - RECOMMEND softmax for the primary GPU pipeline because: (a) no boundary singularity, (b) simpler implementation, (c) already in joint_optimizer.py, (d) well-conditioned for typical LIBS compositions.
       - RECOMMEND ILR for diagnostic/analysis purposes where the Aitchison geometry of compositional data is important (e.g., visualizing composition trajectories, PCA in composition space).
       - Validation criteria:
         * sum(softmax(theta)) = 1 to machine precision for random theta in [-100, 100]^D
         * ilr_inverse(ilr(C)) = C to machine precision for random C on simplex
         * Jacobian dC/dtheta has exactly D-1 nonzero singular values
         * JAX grad(loss)(theta) matches finite-difference gradient to < 1e-6

    Reference existing implementations in cflibs/inversion/closure.py (ILR, Helmert basis) and cflibs/inversion/joint_optimizer.py (softmax parameterization) for consistency.
