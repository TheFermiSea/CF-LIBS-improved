    Produce a complete LaTeX derivation of the CF-LIBS batch forward model for GPU computation. This derivation composes DERV-01 through DERV-04 into the complete pipeline. Structure:

    1. FORWARD MODEL PHYSICS (1 page):
       - The CF-LIBS forward model maps plasma parameters to a synthetic emission spectrum:
         f: (T, n_e, C_1, ..., C_N; lambda_grid) -> S(lambda)
       - Decompose into four sequential stages:

         STAGE 1 -- Saha ionization balance (from DERV-03):
           Given (T_eV, n_e, C_s) for each element s:
           S_z(T_eV, n_e) = (SAHA_CONST/n_e) * T_eV^{3/2} * (U_{z+1}/U_z) * exp(-chi_z/T_eV)
           Ion fractions: f_I = 1/(1 + S_1 + S_1*S_2), f_II = S_1*f_I, f_III = S_2*f_II
           Stage densities: n_{s,z} = C_s * n_total * f_z(T_eV, n_e)
           NOTE: For the forward model, n_e is an INPUT (not iterated). Anderson acceleration (DERV-03) is used in the inversion to find self-consistent n_e; in the forward model, we simply evaluate the Saha ratios at the given n_e.

         STAGE 2 -- Boltzmann level populations:
           For each spectral line l with upper level k in element s, ionization stage z:
           n_k = n_{s,z} * (g_k / U_z(T_eV)) * exp(-E_k / T_eV)
           This gives the upper-level population for each line.

         STAGE 3 -- Line emissivities:
           epsilon_l = (h * c / (4 * pi * lambda_l)) * A_ki_l * n_k_l  [W m^-3 sr^-1]
           Convert n_k from cm^-3 to m^-3: multiply by 1e6.
           This is evaluated for all N_lines lines simultaneously.

         STAGE 4 -- Voigt broadening and spectrum assembly (from DERV-01):
           For each line l, compute broadening parameters:
           sigma_D_l = (lambda_l / c) * sqrt(k_B * T_K / M_atom_l)  [nm, Doppler]
           gamma_S_l = stark_w_l * (n_e / 1e16)  [nm, Stark HWHM]
           The synthetic spectrum:
           S(lambda) = sum_{l=1}^{N_lines} epsilon_l * V(lambda - lambda_l; sigma_D_l, gamma_S_l)
           Using the broadcasting pattern from DERV-01:
           diff = lambda_grid[:, None] - lambda_l[None, :]  # (N_wl, N_lines)
           profiles = voigt_kernel(diff, sigma_D, gamma_S)   # (N_wl, N_lines)
           S = jnp.sum(epsilon * profiles, axis=1)           # (N_wl,)

       - State dimensional chain explicitly:
         [n_k] = cm^-3 -> m^-3 via *1e6
         [epsilon] = [J] * [m/s] / ([m] * [s^-1] * [m^-3]) = W m^-3 sr^-1
         [V] = nm^-1
         [S] = [epsilon] * [V] = W m^-3 sr^-1 nm^-1

    2. ARRAY SHAPES AND VMAP DECOMPOSITION (1.5 pages):
       a. Define the shape taxonomy:
          - B: batch dimension (number of parameter combinations)
          - N_el: number of elements (typically 3-15)
          - N_st: max ionization stages per element (3: neutral, singly, doubly ionized)
          - N_lines: total number of spectral lines across all elements and stages
          - N_wl: wavelength grid points

       b. Input arrays:
          - T_eV: (B,) -- electron temperature for each batch element
          - n_e: (B,) -- electron density for each batch element
          - C: (B, N_el) -- composition vector for each batch element
          - lambda_grid: (N_wl,) -- shared wavelength grid (NOT batched)
          - Atomic data (static, NOT batched):
            lines_wl: (N_lines,), lines_aki: (N_lines,), lines_ek: (N_lines,),
            lines_gk: (N_lines,), lines_ip: (N_lines,), lines_z: (N_lines,) int,
            lines_el_idx: (N_lines,) int, lines_mass_amu: (N_lines,),
            lines_stark_w: (N_lines,),
            partition_coeffs: (N_el, N_st, 5), ionization_potentials: (N_el, N_st)

       c. Single-spectrum function signature:
          single_spectrum(T_eV, n_e, C, lambda_grid, atomic_data) -> S  # (N_wl,)
          where T_eV is scalar, n_e is scalar, C is (N_el,)

       d. Batch via vmap:
          batch_spectrum = jit(vmap(single_spectrum, in_axes=(0, 0, 0, None, None)))
          batch_spectrum(T_batch, ne_batch, C_batch, lambda_grid, atomic_data) -> (B, N_wl)

       e. Internal array shapes per single spectrum:
          Stage 1 output: ion_fractions (N_el, N_st) -- fraction in each stage
          Stage 2 output: n_k (N_lines,) -- upper-level population per line
          Stage 3 output: epsilon (N_lines,) -- emissivity per line
          Stage 4 intermediate: profiles (N_wl, N_lines) -- the big matrix
          Stage 4 output: S (N_wl,) -- spectrum

       f. Element-to-line mapping: lines_el_idx maps each line to its element index. Use this to gather C[lines_el_idx] to get per-line composition factors. Similarly, partition_coeffs[lines_el_idx, lines_z] gathers per-line partition function coefficients.

    3. MEMORY BUDGET ANALYSIS (1 page):
       a. The dominant memory consumer is the (N_wl, N_lines) profile matrix in Stage 4.
          Per-spectrum memory: N_wl * N_lines * dtype_bytes
          For float64: 10000 * 5000 * 8 = 400 MB
          For float32: 10000 * 5000 * 4 = 200 MB

       b. With vmap over batch B, XLA may or may not materialize all B copies simultaneously.
          WORST CASE (full materialization): B * N_wl * N_lines * dtype_bytes
          For B=64, float64: 64 * 400 MB = 25.6 GB -- EXCEEDS V100S 16 GB!
          For B=64, float32: 64 * 200 MB = 12.8 GB -- fits with 3.2 GB headroom
          For B=32, float64: 32 * 400 MB = 12.8 GB -- fits with 3.2 GB headroom

       c. XLA FUSION OPTIMISTIC CASE: XLA's op fusion can fold the multiply-accumulate
          (sum over lines) into the profile computation, never materializing the full
          (N_wl, N_lines) matrix. In this case, memory is O(B * N_wl + N_lines) rather
          than O(B * N_wl * N_lines). However, this depends on XLA's optimizer decisions
          and CANNOT be guaranteed without profiling.

       d. RECOMMENDED batch sizes for V100S (16 GB HBM2):
          - Conservative (guaranteed fit): B=32 with float64
          - Standard (likely fit): B=64 with float32
          - With gradient checkpointing: B=16 with float64 (autodiff doubles memory)
          - For manifold generation (forward only, no grad): B=64 float32 or B=32 float64

       e. Ancillary memory:
          - Atomic data arrays: ~N_lines * 10 * 4 bytes ~ 200 KB (negligible)
          - Partition coefficients: N_el * N_st * 5 * 4 ~ 1 KB (negligible)
          - Ion fractions: B * N_el * N_st * 8 ~ 15 KB for B=64 (negligible)
          - Line populations: B * N_lines * 8 ~ 2.5 MB for B=64 (negligible)
          - Total ancillary: < 5 MB (negligible vs profile matrix)

       f. Derive max_batch_size formula:
          max_B = floor((GPU_MEM - OVERHEAD) / (N_wl * N_lines * dtype_bytes))
          For V100S: max_B = floor((16e9 - 2e9) / (10000 * 5000 * 8)) = floor(14e9 / 4e8) = 35
          Conservative: use B = 32 for float64, B = 64 for float32.

    4. XLA COMPILATION AND FUSION (0.5 pages):
       - First-call compilation overhead: JIT compiles the full computation graph. For complex models with N_lines > 1000, expect 30-120 seconds compilation. Subsequent calls with same input shapes are instant.
       - Shape polymorphism: JAX traces by shape, not value. Changing B, N_wl, or N_lines triggers recompilation. For manifold generation, fix these at the start.
       - XLA fusion opportunities:
         * Stages 1-3 (populations + emissivities) are element-wise ops on (N_lines,) arrays -- fully fusible into a single XLA kernel.
         * Stage 4 (broadening) is a broadcasting + reduction that XLA may optimize as a custom GEMM-like kernel.
         * The profile (N_wl, N_lines) outer product followed by weighted sum over lines is mathematically equivalent to a matrix-vector multiply: S = profiles @ epsilon. XLA may recognize this pattern.
       - Device placement: all computation runs on GPU. Data transfer happens once at the start (atomic data to device). Per-batch, only the parameter tuple (T, n_e, C) is transferred (negligible: B * (2 + N_el) * 8 bytes).

    5. CONNECTION TO MANIFOLD GENERATION (0.5 pages):
       - The manifold generator creates a parameter grid: T in [T_min, T_max], n_e in [ne_min, ne_max], C on the simplex.
       - Grid construction: meshgrid over T and n_e gives N_T * N_ne points. For each point, sample compositions from a Dirichlet distribution or uniform simplex grid.
       - Total manifold points: N_manifold = N_T * N_ne * N_comp (e.g., 50 * 50 * 1000 = 2.5M).
       - Processing: chunk into batches of size B, vmapped forward model produces (B, N_wl) spectra per chunk.
       - Total computation: N_manifold / B chunks * compilation (once) + N_manifold * N_wl * N_lines FLOPs.
       - Storage: N_manifold * N_wl * dtype_bytes (e.g., 2.5M * 10000 * 4 = 100 GB for float32). Use HDF5 or Zarr with chunked I/O.
       - Reference the existing ManifoldGenerator._compute_spectrum_snapshot in generator.py which already implements a version of this pipeline. The derivation formalizes what it does and identifies the vmap pattern.

    6. LIMITING CASES (0.5 pages):
       - Single line (N_lines = 1): S(lambda) = epsilon_1 * V(lambda - lambda_1; sigma_1, gamma_1). The (N_wl, 1) profile matrix is just a column vector. No sum needed. Verify shape consistency.
       - Single element (N_el = 1): C = [1.0]. The element-to-line mapping selects all lines of that element. Reduces to the standard CF-LIBS forward model for a single element.
       - Batch size B=1: vmap over a single parameter combination is equivalent to the single-spectrum function. No performance benefit but shape consistency must hold.
       - Zero Stark width (gamma_S = 0): The Voigt profile reduces to a pure Gaussian (Doppler broadening only). This is the gamma -> 0 limit from DERV-01.
       - Zero Doppler width (sigma_D = 0): The Voigt profile reduces to a pure Lorentzian (Stark broadening only). This is the sigma -> 0 limit from DERV-01.
       - T -> 0: All populations in ground state. Emissivity is nonzero only for lines with E_k = 0 (ground-state transitions, if any). Most lines vanish.
       - n_e -> 0: Saha ratios S_z -> infinity (fully ionized? No: S_z ~ 1/n_e * exp(-chi/T), so S_z -> infinity means all atoms ionized -- but physically n_e -> 0 means no plasma). This regime is outside the model validity. Flag as undefined.

    7. VALIDATION CRITERIA:
       - Batch vs sequential: |batch_spectrum[i] - single_spectrum(T[i], n_e[i], C[i])| / |single_spectrum| < 1e-12 for all i in 1..B. vmap must NOT change the numerical result.
       - Memory: Actual GPU memory usage (measured via jax.device_memory_usage or nvidia-smi) within 20% of the derived budget.
       - Compilation: JIT compilation succeeds for (B=64, N_wl=10000, N_lines=5000) without OOM.
       - Dimensional chain: verify S(lambda) has units W m^-3 sr^-1 nm^-1 by tracking units through all four stages.
       - Known result: for a single Fe I line at T=1 eV, n_e=1e17, the emissivity and Voigt FWHM should match the existing CPU forward model to < 0.1%.

    Reference the existing implementation in cflibs/manifold/generator.py (ManifoldGenerator._compute_spectrum_snapshot and the vmap/batch pattern) for architectural consistency.
