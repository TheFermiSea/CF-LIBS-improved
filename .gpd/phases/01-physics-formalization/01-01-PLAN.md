  <files>.gpd/research/derivations/derv01_voigt_profile.tex</files>
  <action>
    Produce a complete LaTeX derivation of the Voigt profile for GPU computation. Structure:

    1. PHYSICS MOTIVATION (1 page):
       - Spectral line formation in LTE plasma: Doppler broadening (thermal motion, Gaussian with sigma_D = (lambda_0/c)*sqrt(kT/M_atom)) and Stark+natural broadening (Lorentzian with gamma_Stark = w * n_e/1e16 HWHM).
       - The Voigt profile is the convolution: V(lambda) = integral G(lambda') * L(lambda - lambda') dlambda'.
       - Express via Faddeeva function: V(lambda) = Re[w(z)] / (sigma_G * sqrt(2*pi)) where z = (lambda - lambda_0 + i*gamma) / (sigma_G * sqrt(2)).
       - State ALL conventions explicitly: gamma = Lorentzian HWHM (NOT FWHM), sigma_G = Gaussian std dev (NOT FWHM). Give FWHM conversion formulas.

    2. FADDEEVA FUNCTION ALGORITHMS (2 pages):
       a. Define w(z) = exp(-z^2) * erfc(-iz) for Im(z) >= 0. State the analyticity domain.
       b. Weideman N=36 rational approximation:
          - Moebius transform Z = (L + iz)/(L - iz) mapping upper half-plane to unit disk
          - Optimal L = sqrt(N/sqrt(2)) = 5.0454 for N=36
          - w(z) = (1/sqrt(pi)) * L/(L-iz) + (2/(L-iz)^2) * sum_{n=0}^{N-1} c_n * Z^n
          - State that the 36 coefficients c_n are pre-computed (cite Weideman 1994 Table I; they are already in profiles.py lines 412-449). DO NOT re-derive the coefficients.
          - Emphasize: this is a SINGLE polynomial evaluation -- branch-free, no conditionals. All threads in a GPU warp execute identical instructions.
       c. Humlicek W4 comparison (brief): 4-region rational approximation. Explain WHY it is unsuitable for JAX autodiff: jnp.where evaluates ALL branches during AD, causing exp() overflow in region 4 and NaN gradients. Cite this as the critical pitfall.
       d. Zaghloul 2024 comparison (brief): Chebyshev subinterval approach, ~1e-13 accuracy. More complex to implement than Weideman; marginal accuracy gain. Keep as accuracy REFERENCE only.

    3. ERROR ANALYSIS (0.5 pages):
       - Weideman N=36 in float64: relative error < 1e-13 (cite Weideman Table 1)
       - In float32: coefficient dynamic range (14 orders of magnitude) effectively reduces to N~20, giving 0.1-1% wing errors. Conclude: float64 required for publication-quality results.
       - V100S float64 throughput is 8.2 TFLOPS (1:2 ratio vs float32), making float64 viable.

    4. JAX PARALLELIZATION STRATEGY (1 page):
       - The spectrum is S(lambda) = sum_lines epsilon_l * V(lambda - lambda_l; sigma_l, gamma_l).
       - This is a (N_wl, N_lines) outer product followed by a sum over lines.
       - Broadcasting approach (PREFERRED): diff = wl_grid[:, None] - line_wl[None, :] produces (N_wl, N_lines) array. Evaluate Voigt element-wise, then sum.
       - Alternative vmap approach: vmap over lines, accumulate. Broadcasting is preferred because it maps to a single GEMM-like operation.
       - State the vmap axes explicitly: for batch forward model (Phase 3), the additional batch dimension is vmapped over axis 0 of (T, n_e, C).
       - Memory: (N_wl, N_lines) * 8 bytes (float64) per spectrum. For N_wl=10000, N_lines=5000: 400 MB. This constrains batch size.

    5. LIMITING CASES (0.5 pages):
       - gamma -> 0: z becomes real, w(x) = exp(-x^2), V -> Gaussian G(lambda) = exp(-(lambda-lambda_0)^2 / (2*sigma^2)) / (sigma*sqrt(2*pi)). Verify.
       - sigma -> 0: z -> i*infinity along imaginary axis, V -> Lorentzian L(lambda) = gamma / (pi * ((lambda-lambda_0)^2 + gamma^2)). Verify.
       - |z| -> infinity: w(z) -> i/(sqrt(pi)*z), V -> Lorentzian wings.
       - Voigt FWHM approximation: f_V = 0.5346*f_L + sqrt(0.2166*f_L^2 + f_G^2) (Olivero & Longbothum 1977, 0.02% accuracy). State for use in validation.

    6. VALIDATION CRITERIA:
       - Numerical: |V_Weideman(z) - V_scipy(z)| / |V_scipy(z)| < 1e-6 for 1000 test points in LIBS regime (y in [0.001, 100], x in [-100, 100])
       - Gradient: d/dz w(z) = 2*(i/sqrt(pi) - z*w(z)) (exact relation). Verify JAX grad against this.
       - Integral: integral V(lambda) dlambda = 1 (normalization) to < 1e-10 relative error by numerical quadrature.

    Use the LOCKED conventions from the project: gamma = HWHM, sigma = std dev, lambda in nm, wavelength-domain normalization. Reference the existing implementation in cflibs/radiation/profiles.py for consistency.

  <name>Task 2: Formalize vectorized Boltzmann plot fitting (DERV-02)</name>
  <files>.gpd/research/derivations/derv02_boltzmann_fitting.tex</files>
  <action>
    Produce a complete LaTeX derivation of batched Boltzmann fitting for GPU computation. Structure:

    1. PHYSICS MOTIVATION (0.5 pages):
       - In LTE, upper-level population: n_k = (g_k / U(T)) * n_total * exp(-E_k / (k_B * T)).
       - Line emissivity: epsilon = (hc / (4*pi*lambda)) * A_ki * n_k.
       - Taking ln: ln(I*lambda / (g_k * A_ki)) = -E_k / (k_B * T) + ln(hc * C_s * n_total / (4*pi * U_s(T))).
       - This is y = a + b*x where x = E_k [eV], b = -1/(k_B*T) [eV^-1], a encodes composition.
       - State units: y is dimensionless (ln of a ratio that becomes dimensionless after cancellation). x in eV. Slope b in eV^-1. Verify the dimensional chain explicitly.

    2. WEIGHTED LEAST SQUARES (1 page):
       - Weights w_i = 1/sigma_y_i^2 where sigma_y = sigma_I / I (relative intensity uncertainty propagated through ln).
       - Design matrix X = [1, x_1; 1, x_2; ...; 1, x_N] of shape (N, 2).
       - Weight matrix W = diag(w_1, ..., w_N).
       - Normal equations: (X^T W X) * beta = X^T W y where beta = [a, b]^T.
       - Explicit 2x2 solution:
         S_w = sum(w_i), S_wx = sum(w_i*x_i), S_wy = sum(w_i*y_i),
         S_wxx = sum(w_i*x_i^2), S_wxy = sum(w_i*x_i*y_i).
         det = S_w * S_wxx - S_wx^2.
         b = (S_w * S_wxy - S_wx * S_wy) / det.
         a = (S_wxx * S_wy - S_wx * S_wxy) / det.
       - Temperature: T_K = -1 / (b * k_B_eV) where k_B_eV = 8.617e-5 eV/K.
       - Why normal equations and NOT QR/SVD: for a 2x2 system, the condition number is bounded by the data spread in E_k (typically 0-10 eV), so the normal equations are numerically stable. QR would add 3x overhead for no benefit.

    3. COVARIANCE AND UNCERTAINTY (0.5 pages):
       - Covariance matrix: Cov(beta) = (X^T W X)^{-1}.
       - For 2x2: Cov = (1/det) * [[S_wxx, -S_wx], [-S_wx, S_w]].
       - Uncertainty in temperature: sigma_T = T^2 * k_B_eV * sigma_b where sigma_b = sqrt(Cov[1,1]).
       - Reduced chi-squared: chi^2_red = (1/(N-2)) * sum w_i * (y_i - a - b*x_i)^2.
       - State that chi^2_red >> 1 indicates non-LTE or self-absorption; chi^2_red << 1 indicates overestimated uncertainties.

    4. BATCHED GPU FORMULATION (1 page):
       - Batch dimension B: one Boltzmann fit per (element, ionization stage) per spectrum.
       - For 5 elements, 2 ionization stages each, 100 spectra: B = 1000.
       - Pad-and-mask: x shape (B, N_max), y shape (B, N_max), mask shape (B, N_max) boolean.
       - Masked sums: S_w = sum(w * mask, axis=1), etc. Each is a single jnp.sum over axis 1.
       - Total operations: 5 dot products (S_w, S_wx, S_wy, S_wxx, S_wxy) + 2 divisions per batch element.
       - This is trivially parallel: no cross-batch communication.
       - JAX: @jit applied to the batched function. No vmap needed -- the batch dimension is handled by array broadcasting.
       - Memory: (B, N_max) * 8 bytes * 5 arrays. For B=1000, N_max=50: 2 MB. Negligible on GPU.

    5. OUTLIER REJECTION (0.5 pages):
       - Sigma-clipping: compute residuals r_i = y_i - a - b*x_i. Mask out |r_i| > k*sigma_r (k=2.5 typical). Refit.
       - This is fully vectorizable in JAX: compute all residuals, create boolean mask, refit.
       - RANSAC: NOT recommended for GPU (requires random sampling + early termination, incompatible with JAX JIT).
       - Huber M-estimation: iteratively reweighted least squares (IRLS). Can be done in JAX via jax.lax.scan but adds 5-10 iterations. Use for final refinement only, not batch sweep.

    6. SAHA CORRECTION (0.5 pages):
       - After fitting each ionization stage independently, map ionic Boltzmann intercepts to the neutral reference plane using Saha equation.
       - Intercept correction: a_II_corrected = a_II + ln(n_e / SAHA_CONST * T_K^{-3/2} * U_I/U_II * exp(chi_I/(k_B*T))).
       - This enables a common-slope fit across all stages: the slope (temperature) should be consistent.
       - Multi-element closure then normalizes sum(C_s) = 1 (see Plan 01-02, DERV-04).

    7. VALIDATION CRITERIA:
       - Synthetic test: generate Boltzmann data from known T, add Gaussian noise, recover T within 2*sigma_T.
       - Uniform weights (w_i = 1): must reduce to ordinary least squares.
       - Single element: must reproduce standard textbook Boltzmann plot slope.
       - Degenerate case: all E_k equal -> det = 0, must be diagnosed (not NaN).

    Reference the existing implementation in cflibs/inversion/boltzmann.py for conventions and ensure the derivation is consistent with the codebase.
  </action>

    2. Limiting cases: (a) w_i = 1 for all i reduces to OLS formulas, (b) N=2 gives exact fit with zero residual, (c) all x_i equal gives det=0 (degenerate).
    3. Covariance matrix: (X^T W X)^{-1} is 2x2, positive definite, with correct units: Cov[0,0] dimensionless, Cov[1,1] in eV^{-2}.
    4. Batch formulation: shapes are explicitly stated for all arrays. vmap axes identified.
