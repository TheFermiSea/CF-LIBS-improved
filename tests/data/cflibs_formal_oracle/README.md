# Verified-spec regression oracle (vendored from `cflibs-formal`)

`fixtures.json` is generated by the **machine-verified** Lean 4 + mathlib CF-LIBS spec
`cflibs-formal` — a companion project that proves the CF-LIBS forward map, inverse problem,
identifiability, and reliability bounds, all axiom-clean (`{propext, Classical.choice,
Quot.sound}`, no `sorry`). Each fixture **check instantiates a proven theorem**; the corpus
passed a whole-corpus literature-validity audit (0 divergent, 0 unverified citations).

These fixtures let `tests/inversion/physics/test_oracle_regression.py` regression-test this
pipeline's **real** physics functions against the verified spec, so a refactor that silently
breaks a formula/sign/factor is caught.

## Conventions (read before using)

- **Dimensionless inputs.** The spec is dimensionless: `kB = T = Fcal = 1`, and `E` is in units
  of `kB·T`. To drive this pipeline (which is physical — Kelvin/eV), the test maps a dimensionless
  energy `E` to eV at a reference temperature `T_ref` via `E_eV = E * KB_EV * T_ref` (so
  `exp(-E_eV / (KB_EV * T_ref)) = exp(-E)` reproduces the spec's Boltzmann factor). Wavelengths
  are held constant so the energy ordinate `ln(I*lambda/(gA))` shares the spec's reduced-ordinate
  slope.
- **`Float != R`.** The spec is real-valued and noncomputable; the fixtures come from a `Float`
  mirror, so checks are tolerance-based (`rtol ~ 1e-6`) — ample to catch formula/sign/factor bugs,
  not a bit-exact comparison.

## Scenarios

| `kind` | what it pins |
| --- | --- |
| `multi-element-composition` | ternary alloy, 3 distinct elements (own `g,E,A,U_s`), 4 lines each, distinct optical depths: forward intensities, partition functions, two-line temperature, OLS, self-absorption, closure (`sum C = 1`), calibration-free |
| `saha-boltzmann` | neutral+ion stages: recover `T` and electron density `n_e` |
| `error-budget` | the **derived** reliability thresholds (`requiredEnergySpread`, `maxPerLineError`, ...) with tightness invariants — mirrored in `cflibs/inversion/physics/error_budget.py` |
| `energy-ordinate` | distinct per-line `lambda`: the `ln(I*lambda/gA)` (energy) and `ln(I/gA)` (photon-rate) conventions agree |

## Regenerating

From the `cflibs-formal` checkout:

```bash
lake exe oracle-fixtures > oracle/fixtures.json
cp oracle/fixtures.json <this-repo>/tests/data/cflibs_formal_oracle/fixtures.json
```

Keep this copy in sync when the spec's oracle changes. The committed copy is the regression anchor.
